# 報帳系統功能要求與測試清單

## 一、需求演進摘要

### A. 原始基本功能
- 入口頁選擇身份
- 支出報帳新增、編輯、草稿、送出、刪除/作廢、全部資料
- 出差報帳新增、編輯、草稿、送出、刪除/作廢、全部資料
- PDF 匯出
- 附件上傳
- 簽名檔上傳
- Excel 下載
- 本機快取與待同步佇列

### B. 後續修正需求
- 支出與出差改為 master dataframe / 單次載入共用
- 草稿送出後，前端、雲端、Excel 狀態一致
- 降低切頁與操作等待時間
- 出差交通方式下方直接顯示四個欄位
- 支出與出差共用 plan code，並支援「其他」輸入
- admin 可永久移除單筆資料
- hard delete 前先備份到 deleted archive
- hard delete 需二次確認且可取消
- 雲端表單與下載 Excel 都要顯示 deleted / void
- 雲端分頁不可空白
- 重跑 / 重部署 / 重開後不得遺失既有資料
- deleted archive 可一鍵還原

---

## 二、測試前置條件
- 已設定至少 1 位 admin、1 位一般 user
- 已有 Apps Script / 雲端表單可用
- `shared_plan_options.py` 已配置 plan code
- 本機 `data/cache/` 可寫入
- 至少準備以下測試資料：
  - 1 筆支出 draft
  - 1 筆支出 submitted
  - 1 筆出差 draft
  - 1 筆出差 submitted

---

## 三、身份與權限

### 3.1 入口與身份
- [ ] 可從入口頁切換不同身份
- [ ] 切換身份後，姓名 / Email / 角色正確顯示
- [ ] user 看不到 admin 專屬永久移除功能
- [ ] admin 可看到永久移除功能

### 3.2 權限隔離
- [ ] user 只能編輯自己的 draft
- [ ] user 不可永久移除他人資料
- [ ] admin 可瀏覽全部資料
- [ ] admin 可執行永久移除與 archive 還原

## 七、Excel / 雲端表單一致性

- [ ] `草稿列表` 包含 draft + deleted

### 7.2 出差

- [ ] `申請列表` 包含 submitted + void

## 八、永久移除（hard delete）

### 8.1 支出
- [ ] admin 可看到「移除單筆資料」
- [ ] 第一次點擊後只進入確認狀態，不立即刪除
- [ ] 點「確認永久移除」才真的刪除
- [ ] 點「取消」後不執行刪除
- [ ] 刪除前會寫入 deleted archive
- [ ] 前端列表立即消失
- [ ] 重新整理後不會再出現

### 8.2 出差
- [ ] admin 可看到「移除單筆資料」
- [ ] 第一次點擊後只進入確認狀態，不立即刪除
- [ ] 點確認後才真的刪除
- [ ] 點取消後不執行刪除
- [ ] 刪除前會寫入 deleted archive
- [ ] 前端列表立即消失
- [ ] 重新整理後不會再出現

---

## 九、deleted archive 一鍵還原

### 9.1 支出
- [ ] sidebar 可看到 deleted archive 還原區
- [ ] 可列出尚未還原的 archive 項目
- [ ] 點「一鍵還原支出紀錄」可恢復資料
- [ ] 原 submitted / void 紀錄還原為 submitted
- [ ] 原 draft / deleted 紀錄還原為 draft
- [ ] 還原後前端列表可看見資料
- [ ] 還原後 archive 項目標記為已還原，不再重複出現在可還原清單

### 9.2 出差
- [ ] sidebar 可看到 deleted archive 還原區
- [ ] 可列出尚未還原的 archive 項目
- [ ] 點「一鍵還原出差紀錄」可恢復資料
- [ ] 原 submitted / void 紀錄還原為 submitted
- [ ] 原 draft / deleted 紀錄還原為 draft
- [ ] 還原後前端列表可看見資料
- [ ] 還原後 archive 項目標記為已還原，不再重複出現在可還原清單

## 十、資料保留與重啟穩定性
- [ ] 重開 Streamlit 後既有資料仍可讀
- [ ] 重部署後既有本機 cache 不被清空
- [ ] 重部署後既有雲端資料仍可讀
- [ ] 前端看不到資料時，確認不是因 sheet 名稱變更造成
- [ ] 下載 Excel 與前端資料筆數一致
- [ ] 雲端表單與前端資料筆數一致

## 十一、同步與快取
